# Security Readiness

This checklist evaluates whether a Solana project is secure enough for public launch.

## Access Control

### Upgrade Authority

* Upgrade authority identified
* Upgrade authority documented
* Authority ownership reviewed
* Authority risks documented

### Multisig

* Multisig configured
* Signers documented
* Emergency procedures documented

## Key Management

### Secrets

* Private keys not stored in repository
* Secrets stored securely
* Access permissions reviewed

### Recovery

* Recovery procedures documented
* Backup access tested

## Dependency Review

### Third Party Dependencies

* External programs reviewed
* SDK versions documented
* Known vulnerabilities reviewed

## Audit Readiness

### Security Review

* Security assessment completed
* Critical findings addressed
* Remaining risks documented

## Operational Security

### Incident Preparedness

* Security incident owner assigned
* Communication plan documented
* Escalation process documented
